# 【Hello AI 科技致善】Soul Soother——用抽象接住崩溃，用玩笑降低痛感

## 一、摘要

面向极端情绪状态人群（自杀意念高发群体），在深夜情绪崩溃、想死念头涌现时，提供一个"随手打开笑一笑"的轻量级趣味治愈网站。通过 SBTI 式抽象测评、发疯文学、互助卡、求助贴、激励弹幕和治愈内容合集，让用户在会心一笑中获得短暂喘息。目前通过模拟测试验证，核心功能全部可用，页面加载 < 2秒。

## 二、真实场景与需求

### 目标人群

- **自杀意念高发群体**：经常有"不想活了"念头的人
- **边缘型人格特质人群**：情绪极端波动，自我认同混乱
- **重度抑郁/双相情感障碍患者**：长期与死亡念头共存
- **自伤行为人群**：通过自伤缓解情绪痛苦
- **社会孤立者**：长期缺乏社交连接，感到被世界抛弃

### 痛点描述

**场景一：凌晨3点的"不想活了"**
> 小李凌晨3点醒来，脑子里只有一个念头："活着真没意思"。热线要说话（不想说话）、医生要预约（太麻烦）、正能量内容让人更烦（"加油""振作"会让他更想死）。他打开 Soul Soother，看到一个测试"测测你的精神状态有多美丽"，题目包括"你最近一次崩溃是因为？A. 外卖迟到了 B. 发现自己还活着 C. 以上都是"。他笑了，然后哭了，然后觉得"好像还能再活一天"。

**场景二：自伤前的"缓冲"**
> 小张拿起刀片前，习惯性地打开 Soul Soother。今天推荐的是一个"发疯文学生成器"，输入"我想死"，生成了一段："我想死，但我的快递还没到，我的剧还没追完，我的猫还没喂，所以我决定先活着，等这些都做完了再死——结果发现，永远做不完，所以我永远死不了，真抽象。"他放下刀片，笑了。

**场景三：情绪崩溃后的"自我嘲讽"**
> 小王刚大哭完一场，眼睛肿得像核桃。他打开 Soul Soother 的"今日抽象"，看到一句话："我的精神状态：一边想死，一边怕死，一边觉得自己不配死，一边觉得死了也挺好，最后决定先活着，因为死了就看不到这么抽象的自己了。"他截图发朋友圈，收获了一堆"哈哈哈"，突然觉得没那么孤单了。

### 现有做法与不足

| 现有方式 | 不足之处 |
|----------|----------|
| 心理咨询 | 太贵、太正式、需要预约，低能量状态下难以启动 |
| 心理援助热线 | 需要说话，很多人不想说话；等待时间长 |
| 正能量内容 | "加油""振作"让抑郁人群更反感，增加 guilt |
| 传统心理测评 | 太严肃、太死板，像在做题，增加压力 |
| 社交媒体 | 算法推荐负面内容，社交比较有压力 |
| 朋友安慰 | "想开点""别多想"反而让人更难受 |

## 三、作品介绍

### 产品形态
Web 应用（纯前端，零框架，极致轻量）

### 核心功能

| 功能 | 说明 | 解决什么问题 |
|------|------|-------------|
| 📝 今日抽象 | 每日更新发疯文学/抽象文案 | 让用户在第一眼就笑出声 |
| 🧠 趣味测评 | SBTI 式抽象测评，荒诞题目 | 不是做题，是和一个懂梗的人聊天 |
| 📝 发疯文学生成器 | 输入情绪关键词生成发疯文学 | 把"想死"变成"想死但快递还没到" |
| 💡 "想死"替代方案 | 生成"先做完XX再死"的荒谬列表 | 用荒谬延长时间，降低冲动 |
| 💝 互助卡 | 抽取温暖但不鸡汤的互助卡片 | 给用户一个被拥抱的感觉 |
| 🆘 求助贴/互助区 | 轻量级发帖，一键"抱抱" | 让用户知道"我不是一个人" |
| 💬 激励弹幕 | 滚动播放抽象式鼓励 | 营造"有人陪着你"的氛围 |
| 🎮 治愈合集 | 收录低能量友好的小游戏/网页 | 提供具体可做的放松内容 |
| 🌙 深夜急救包 | 凌晨1-5点专属内容 | 在最危险的时刻提供陪伴 |

### 设计特点

- **零门槛**：无需注册、无需登录、打开即用
- **低认知负荷**：大按钮、少选项、清晰层级
- **无压迫感**：没有待办、没有进度、没有通知
- **抽象语言**：用 Z 世代最懂的梗，不是"加油"是"阴暗爬行也算移动"
- **安全边界**：明确免责声明，危机内容自动引导到求助资源

## 四、用 SOLO 实现的过程

### 需求拆解

```
极端情绪人群的需求
    ↓
核心问题：凌晨想死时不知道能干嘛 + 讨厌被教育 + 需要被懂
    ↓
解决思路：用他们自己的语言（抽象/梗）+ 提供即时可用的内容 + 营造陪伴感
    ↓
产品形态：轻量级 Web 应用
    ↓
关键能力：内容创作（发疯文学）+ 测评设计（SBTI 风格）+ 交互设计（低能量友好）
```

### 使用的 SOLO 能力

| SOLO 能力 | 应用场景 |
|-----------|----------|
| 内容生成 | 创作发疯文学、抽象文案、测评题目 |
| 用户体验设计 | 设计低能量友好的交互流程 |
| 代码生成 | 生成 HTML/CSS/JS 代码 |
| 多文件管理 | 管理多个页面和组件 |

### 关键 Prompt 示例

**Prompt 1：发疯文学创作**

```
你是一个精通 Z 世代网络文化的文案创作者。请创作 10 条"发疯文学"风格的文案，面向抑郁/情绪低落人群。

要求：
- 使用自嘲、黑色幽默的风格
- 包含"想死""崩溃""孤独"等情绪，但用荒谬的方式消解
- 参考风格："我想死，但我的快递还没到"
- 每条 30-80 字
- 让人看了会会心一笑，而不是更难受
```

**Prompt 2：测评题目设计**

```
请设计一个趣味心理测评，主题是"测测你的精神状态有多美丽"。

要求：
- 参考 SBTI 风格，题目要像段子
- 选项要有梗，不能只是为了搞笑
- 背后对应维度：社交距离、行动力、情绪低落感、信任感
- 题目示例："你最近一次崩溃是因为？A. 外卖迟到了 B. 发现自己还活着 C. 以上都是 D. 我不崩溃，我直接发疯"
- 结果类型：正常人类、抽象艺术家、发疯大师、传说级疯批
- 结果描述要用冒犯式总结 + 温柔建议
```

**Prompt 3：互助卡文案**

```
请创作 15 张"互助卡"文案，面向情绪低落/抑郁人群。

要求：
- 温暖但不鸡汤
- 不说"加油""振作""想开点"
- 可以说"先喝水""允许自己不营业""阴暗爬行也算移动"
- 每张 10-20 字
- 让人感觉到被理解，而不是被教育
```

### 开发过程

1. **Day 1 上午**：用 SOLO 完成需求分析、PRD、设计风格调研
2. **Day 1 下午**：用 SOLO 生成核心页面代码（首页、测评、工具、合集、关于）
3. **Day 2 上午**：填充内容数据（发疯文学、测评题目、互助卡、治愈合集）
4. **Day 2 下午**：测试优化，添加深夜急救包、弹幕、求助贴等功能
5. **Day 3**：部署到 GitHub Pages，撰写参赛帖子

### 踩过的坑

1. **内容尺度**：发疯文学容易过界，需要反复调整"自嘲"和"轻生"的边界。最终原则是：可以拿"想死"开玩笑，但始终暗示"活着也挺好"。
2. **危机处理**：求助贴功能需要识别危机内容并引导到专业资源。实现了关键词检测 + 弹窗提示。
3. **性能优化**：纯前端项目，需要确保加载速度。通过 CSS 变量、精简代码、无外部依赖实现 < 2秒加载。
