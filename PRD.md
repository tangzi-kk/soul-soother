# Soul Soother - 产品需求文档

## 1. 项目概述

### 1.1 产品名称
**Soul Soother**（心灵抚慰者）

### 1.2 一句话描述
一个面向极端情绪状态人群的轻量级情绪陪伴网站，通过引力场词汇交互、对话式检测、陪伴社区等功能，为用户提供温暖的情绪缓冲空间。

### 1.3 核心定位
- **不是医学诊断工具**：明确免责声明，不做心理评估或治疗建议
- **是情绪缓冲站**：提供安全的情绪释放空间
- **是陪伴者**：让用户感受到"你不是一个人"
- **是缓一缓的角落**：帮助用户在极端情绪中暂停、呼吸、重新找回平衡

### 1.4 目标用户
- 情绪极端波动人群
- 深夜失眠、情绪低落人群
- 感到孤独、需要陪伴的人群
- 正在经历焦虑、抑郁情绪的人群

### 1.5 参赛赛道
「Hello AI 科技致善」公益赛道 - 心理健康陪伴

## 2. 需求分析

### 2.1 用户痛点

| 痛点 | 具体场景 | 现有解决方案不足 |
|------|----------|-----------------|
| 极端情绪下不知道该做什么 | 凌晨3点，躺在床上，脑子里全是负面念头 | 热线要说话、医生要预约、正能量内容让人更烦 |
| 需要被"看见"而不是被"教育" | 想找人倾诉，但讨厌被说教 | 传统心理支持太正式、太说教 |
| 需要简单、无压力的交互 | 不想填写复杂问卷、不想与人对话 | 现有工具太复杂、门槛高 |
| 深夜是最危险的时候 | 凌晨1-5点，情绪最脆弱 | 深夜没有人在线，没有资源可用 |
| 需要即时的情绪缓解 | 情绪上来时需要立刻有东西可以做 | 没有随手可用的情绪调节工具 |

### 2.2 用户场景

**场景一：凌晨3点的情绪漩涡**
> 小李凌晨3点醒来，脑子里全是负面念头。他打开 Soul Soother，看到首页漂浮着各种词汇，他开始点击积极词汇"呼吸"、"陪伴"、"存在"，看着这些词汇汇聚到屏幕中央，心里慢慢平静下来。

**场景二：情绪崩溃前的缓冲**
> 小张感觉情绪快要崩溃，他打开"缓一缓"页面，选择"喘口气"功能，跟着呼吸气泡一起深呼吸，30秒后感觉平复了一些。

**场景三：感到孤独时的陪伴**
> 小王一个人在家，感到非常孤独。他打开"陪伴"页面，抽了一张陪伴卡，上面写着"你值得被好好对待"，他又看了看深夜弹幕墙，看到很多人和他一样在深夜醒着，感觉没那么孤单了。

**场景四：需要紧急帮助时**
> 小赵感到非常难受，他点击首页的紧急入口，进入"缓冲站"页面，选择了中度危机的缓冲卡，照着上面的建议做，感觉好了一些。

## 3. 核心设计理念：温柔的陪伴

### 3.1 设计原则
- **无压力交互**：不需要注册、不需要登录、不需要填写任何信息
- **温柔不说教**：不说"加油""振作"，只说"我在""慢慢来"
- **可视化疗愈**：通过视觉交互帮助用户调节情绪
- **即时反馈**：每一个动作都有回应，让用户感到被重视

### 3.2 语言风格示例

| 场景 | 传统方式（❌） | Soul Soother 方式（✅） |
|------|--------------|------------------------|
| 用户说"我想死" | "不要这样想，生命很宝贵" | "我知道你很难受，先缓一缓，我们一起找办法" |
| 用户情绪崩溃 | "深呼吸，放松" | "崩溃也没关系，先喘口气，慢慢来" |
| 用户感到孤独 | "你并不孤单" | "深夜里，还有很多人和你一样醒着" |
| 深夜陪伴 | "晚安，好梦" | "晚安，希望你能好好休息，明天再面对" |

## 4. 功能规划

### 4.1 页面结构

```
Soul Soother
├── 首页 (Home)           # 引力场词汇交互
├── 快速检测 (Checkin)    # 小缓对话式情绪检测
├── 陪伴 (Mutual)         # 陪伴卡、弹幕墙、留言
├── 缓冲站 (Kit)          # 呼吸空间、缓冲卡、资源
└── 缓一缓 (Play)         # 喘口气、抓回现实、等一下、找理由、倒垃圾、翻译情绪、小游戏
```

### 4.2 核心功能详解

#### 4.2.1 首页 - 引力场词汇交互
- **背景词汇**：漂浮的词汇分为积极词汇（彩色）和丧词汇（灰色）
- **点击积极词汇**：词汇被吸引到屏幕中央，产生粒子特效，累计连击显示
- **点击丧词汇**：词汇淡出消失，在原地生成一个积极词汇替代
- **紧急入口**：底部固定的紧急入口，随时可进入缓冲站
- **左滑提示**：提示用户可以左滑切换页面

#### 4.2.2 快速检测 - 小缓对话式
- **对话形式**：AI角色"小缓"以聊天形式进行情绪检测
- **问题类型**：选择题、滑动条评分
- **即时反馈**：根据用户回答给出温柔回应
- **结果建议**：检测结束后提供个性化建议

#### 4.2.3 陪伴页面
- **陪伴卡抽取**：点击随机获取一张温暖的陪伴卡片
- **深夜弹幕墙**：实时滚动的匿名留言
- **留一句给陌生人**：用户可以发布匿名留言
- **AI回复**：用户发布后可能收到AI的温暖回复

#### 4.2.4 缓冲站页面
- **30秒呼吸空间**：倒计时呼吸练习，跟着数字倒数
- **缓冲卡**：三个等级（轻度/中度/高危）的应对建议
- **温暖资源**：心理援助热线、医院联系方式

#### 4.2.5 缓一缓页面
- **喘口气**：呼吸动画引导深呼吸
- **抓回现实**：五感 grounding 练习（看到3样东西、听到2种声音、摸到1个东西）
- **等一下**：延迟决定清单，让用户先完成一些小事
- **找理由**：留恋卡片，帮助用户找到活下去的理由
- **倒垃圾**：发疯文学生成器，输入烦恼生成发疯文案
- **翻译情绪**：把坏念头翻译成真实需求
- **玩个小游戏**：推荐放松小游戏

## 5. 技术方案

### 5.1 技术栈
- **前端**：HTML5 + CSS3 + JavaScript（纯原生，零框架）
- **动画**：p5.js（引力场词汇效果）
- **AI集成**：支持多种AI模型（NVIDIA、通义千问、Kimi等）
- **存储**：localStorage（对话记忆、配置）
- **音效**：Web Audio API

### 5.2 项目结构

```
soul-soother/
├── index.html              # SPA 单页应用入口
├── css/
│   └── style.css           # 完整样式文件
├── js/
│   ├── app.js              # 主应用逻辑
│   ├── data.js             # 内容数据
│   ├── gravity.js          # 引力场词汇逻辑
│   └── p5-gravity.js       # p5.js 引力场实现
├── audio/                  # 音效文件
└── README.md
```

### 5.3 核心技术亮点

#### 引力场词汇系统
- 使用 p5.js 实现物理模拟
- 词汇分为积极和消极两类
- 点击积极词汇产生吸引效果和粒子特效
- 点击消极词汇使其淡出并生成积极词汇替代

#### 对话式检测系统
- 支持多轮对话
- 支持选择题和滑动条两种交互方式
- 集成AI实现智能回复（带超时降级）

#### 音效系统
- 使用 Web Audio API 生成柔和音效
- 支持开关和音量调节
- 不同交互有不同音效反馈
