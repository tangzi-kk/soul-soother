# 【Hello AI 科技致善】Soul Soother——用抽象接住崩溃，用玩笑降低痛感

## 一、摘要

面向极端情绪状态人群，在深夜情绪崩溃、想死念头涌现时，提供一个"随手打开笑一笑"的轻量级趣味治愈网站。通过 p5.js 语义引力场、状态导向交互和 Z 世代抽象自嘲语言，让用户在会心一笑中获得短暂喘息，将极端情绪缓一缓。

在线体验：https://tangzi-kk.github.io/soul-soother/
开源仓库：https://github.com/tangzi-kk/soul-soother

---

## 二、真实场景与需求

### 目标人群

- **自杀意念高发群体**：经常有"不想活了"念头的人
- **边缘型人格特质人群**：情绪极端波动，自我认同混乱
- **重度抑郁/双相情感障碍患者**：长期与死亡念头共存
- **自伤行为人群**：通过自伤缓解情绪痛苦
- **社会孤立者**：长期缺乏社交连接，感到被世界抛弃

### 痛点描述

**场景一：凌晨3点的"不想活了"**
> 小李凌晨3点醒来，脑子里只有一个念头："活着真没意思"。热线要说话（不想说话）、医生要预约（太麻烦）、正能量内容让人更烦（"加油""振作"会让他更想死）。他打开 Soul Soother，首页是一个文字引力场——屏幕上漂浮着"你不是一个人""撑不住就躺会儿"等文字，他随手一点，周围的文字被引力吸引聚拢，像在回应他。他点了"快速检测"，和一个叫"小缓"的AI聊了几句，结果是"你现在像低电量手机，需要充电但不是紧急情况"。他笑了，然后觉得"好像还能再活一天"。

**场景二：自伤前的"缓冲"**
> 小张拿起刀片前，习惯性地打开 Soul Soother。他直接点进"缓冲站"，启动30秒呼吸空间，跟着数字倒数。呼吸结束后，他点开"缓冲卡"——中度档，上面写着"离开当前环境，去安全的地方""拨打心理援助热线 400-161-9995"。他放下刀片，拨了电话。

**场景三：情绪崩溃后的"自我嘲讽"**
> 小王刚大哭完一场，打开 Soul Soother 的"缓一缓"页面，点了"倒垃圾"，输入"崩溃"，生成了一段："崩溃就崩溃吧谁还没个发疯的时候啊啊啊啊啊我精神状态很好啊哈哈哈哈哈哈世界毁灭吧但先让我躺会儿"。他截图发朋友圈，收获了一堆"哈哈哈"，突然觉得没那么孤单了。

### 现有做法与不足

| 现有方式 | 不足之处 |
|----------|----------|
| 心理咨询 | 太贵、太正式、需要预约，低能量状态下难以启动 |
| 心理援助热线 | 需要说话，很多人不想说话；等待时间长 |
| 正能量内容 | "加油""振作"让抑郁人群更反感，增加 guilt |
| 传统心理测评 | 太严肃、太死板，像在做题，增加压力 |
| 社交媒体 | 算法推荐负面内容，社交比较有压力 |
| 朋友安慰 | "想开点""别多想"反而让人更难受 |

## 三、作品介绍

### 产品形态
单页 Web 应用（SPA），纯前端，零框架，极致轻量

### 架构设计

Soul Soother 采用**状态导向**的 SPA 五页架构，不是按功能模块划分，而是按用户情绪状态引导：

```
首页（引力场）→ 快速检测 → 陪伴 → 缓冲站 → 缓一缓
    ↑              ↓          ↓        ↓         ↓
  沉浸入口      状态识别    情感陪伴   危机缓冲   自助工具
```

### 核心功能

| 页面 | 功能 | 解决什么问题 |
|------|------|-------------|
| 🏠 首页 | p5.js 语义引力场 + 紧急入口 + 免责声明 | 让用户在第一秒就被"接住"，无需思考就能开始交互 |
| 💓 快速检测 | "小缓"对话式状态检测 | 不是做题，是和一个懂梗的人聊天，快速识别当前状态 |
| 🤝 陪伴 | 陪伴卡抽取 + 深夜弹幕墙 + 留一句给陌生人 + AI回复 | 让用户知道"我不是一个人"，获得被理解的感受 |
| 🛟 缓冲站 | 30秒呼吸空间 + 缓冲卡（轻度/中度/高危）+ 温暖资源 | 在最危险的时刻提供缓冲，降低冲动 |
| 🎈 缓一缓 | 喘口气/抓回现实/等一下/找理由/倒垃圾/翻译情绪/小游戏 | 7种自助工具，根据"我现在想..."快速选择 |

### 缓一缓页面详解

"缓一缓"是 Soul Soother 最核心的创新页面，采用"我现在想..."的快速选择模式：

| 工具 | 说明 | 原理 |
|------|------|------|
| 喘口气 | 跟着圆圈呼吸的呼吸练习 | 腹式呼吸激活副交感神经，降低心率 |
| 抓回现实 | 3-2-1 感官着陆法（看3样/听2种/摸1个） | 将注意力从内在痛苦拉回当下 |
| 等一下 | 生成"先做完XX再死"的荒谬列表 | 用荒谬延长时间，降低冲动 |
| 找理由 | 抽取"还有什么舍不得"的依恋卡 | 激活依恋系统，找到活下去的理由 |
| 倒垃圾 | 输入情绪关键词生成发疯文学 | 把"想死"变成"想死但快递还没到" |
| 翻译情绪 | 坏念头翻译机（"想死"→"先离开当前场景，去倒一杯水"） | 将极端念头翻译成具体可执行的行为 |
| 玩个小游戏 | 推荐治愈网页和小游戏 | 提供具体可做的放松内容 |

### 设计特点

- **零门槛**：无需注册、无需登录、打开即用
- **低认知负荷**：大按钮、少选项、"我现在想..."快速选择
- **无压迫感**：没有待办、没有进度、没有通知、没有社交比较
- **状态导向**：不是功能列表，是情绪状态的引导路径
- **抽象语言**：用 Z 世代最懂的梗，不是"加油"是"阴暗爬行也算移动"
- **安全边界**：免责声明、缓冲卡分级、危机内容引导到求助资源

### 彩蛋系统

Soul Soother 内置了多层彩蛋系统，在用户最意想不到的时刻提供温暖和惊喜：

**1. 首页连击系统**
- 在首页引力场中连续点击彩色词汇，触发连击反馈
- 连击文案递进："抓住了一点光" → "又抓住一点" → "光越来越多了" → "你在发光" → "黑暗在退散" → "世界变亮了" → "光之使者"
- 每次点击生成粒子特效，连击越高粒子越多
- 点击灰色丧词汇则淡出，被积极词汇替代

**2. "想死"替代方案彩蛋**
- 全局随机触发（每次点击 5% 概率，深夜 15% 概率）
- 弹出"等等！"弹窗，用可爱角色提供"先做完XX再死"的替代方案
- 15种主题彩蛋：猫咪安慰🐱、狗狗鼓励🐶、美食诱惑🍜、睡眠召唤💤、音乐疗愈🎵、自然召唤🌿、朋友召唤👥、游戏诱惑🎮、洗澡召唤🚿、被窝召唤🛏️、哭出来😭、发泄一下😤、希望🌈、未来🔮、毛孩子🐾
- 每个彩蛋包含4个可勾选的替代方案 + 紧急热线信息
- 弹窗带粒子特效 ✨💫⭐🌟💕 + 震动动画

**3. 时间敏感彩蛋**
- 根据当前时间段自动匹配不同彩蛋：
  - 深夜（22:00-06:00）：猫头鹰🦉"这个点还没睡的人，都是心里有事的" + 星星⭐"我都没睡，你急什么？"
  - 凌晨（00:00-05:00）：幽灵👻"这个点还在游荡的，除了我，就是你了"
  - 早晨（06:00-09:00）：朝阳🌅"我每天都升起，你也可以"
  - 工作时间（09:00-18:00）：咖啡☕"没有我，你怎么撑到下班？"
  - 傍晚（18:00-22:00）：夕阳🌇"我每天都要落下，但明天还会升起"
- 特殊日期彩蛋：生日🎂"你又活了一年，真厉害！"、新年🎆"新的一年，新的摆烂！"

**4. 隐藏入口**
- 连续点击"Built with TRAE SOLO" 10次 → 打开 AI 模型设置面板（可切换 LLM 提供商）
- 长按"Built with TRAE SOLO" 800ms → 打开对话记忆管理（下载/清空小缓对话记录）

## 四、用 SOLO 实现的过程

### 需求拆解

```
极端情绪人群的需求
    ↓
核心问题：凌晨想死时不知道能干嘛 + 讨厌被教育 + 需要被懂
    ↓
解决思路：用他们自己的语言（抽象/梗）+ 提供即时可用的内容 + 营造陪伴感
    ↓
产品形态：状态导向 SPA
    ↓
关键能力：内容创作（发疯文学）+ 交互设计（状态导向）+ 视觉创新（语义引力场）
```

### 使用的 SOLO 能力

| SOLO 能力 | 应用场景 |
|-----------|----------|
| 内容生成 | 创作发疯文学、抽象文案、测评题目、陪伴卡文案 |
| 用户体验设计 | 设计状态导向的交互流程、低能量友好的界面 |
| 代码生成 | 生成 HTML/CSS/JS 代码、p5.js 引力场 |
| 多文件管理 | 管理单页应用的多页面组件和数据 |
| 算法设计 | 设计语义引力场的粒子系统、流场动力学、轨道力学 |

### 关键 Prompt 示例

**Prompt 1：语义引力场设计**

```
请设计一个 p5.js 文字引力场效果，作为心理健康网站的首页背景。

要求：
- 画布上漂浮着温暖的中文文字粒子（如"你不是一个人""撑不住就躺会儿"）
- 用户点击时创建引力井，周围文字被吸引形成"语义星座"
- 被吸引的文字放大、变色（从灰色到绿色渐变），象征"抓住一点光"
- 引力井衰减后，文字以弹性缓动回归原位
- 背景流场由 Perlin 噪声驱动，即使无交互也保持微妙动态
- 配色：可可棕(#80705f)粒子 + 绿色渐变(#63a648→#8ec492→#e6efb3)强调
```

**Prompt 2：状态导向交互设计**

```
请设计一个"缓一缓"页面，面向情绪崩溃/想死的人群。

要求：
- 顶部是"我现在想..."的快速选择区，包含：喘口气/抓回现实/等一下/找理由/倒垃圾/翻译情绪/玩个小游戏
- 点击后展开对应的工具区域
- "抓回现实"实现 3-2-1 感官着陆法（看3样东西/听2种声音/摸1个东西），可勾选
- "翻译情绪"是坏念头翻译机，输入"想死"→输出"先离开当前场景，去倒一杯水"
- 所有交互必须低认知负荷，大按钮、少文字、无压迫感
```

**Prompt 3：小缓对话式检测**

```
请设计一个对话式的快速状态检测，角色名叫"小缓"。

要求：
- 不是传统问卷，是对话形式，像和朋友聊天
- 3-5个问题，覆盖：自伤意念、社交支持、当前情绪
- 结果分为三级：轻度（像低电量手机）、中度（像系统过热）、高危（不适合一个人硬扛）
- 结果页提供对应的行动建议和推荐功能
- 语气温暖不鸡汤，像"你最近可能有些疲惫"而不是"你有抑郁倾向"
```

### 开发过程

1. **Day 1 上午**：用 SOLO 完成需求分析、PRD、设计风格调研
2. **Day 1 下午**：用 SOLO 生成核心页面代码（首页引力场、快速检测、陪伴、缓冲站、缓一缓）
3. **Day 2 上午**：填充内容数据（发疯文学、测评题目、陪伴卡、深夜弹幕、治愈合集）
4. **Day 2 下午**：实现小缓对话系统、呼吸练习、感官着陆法、坏念头翻译机
5. **Day 3**：部署到 GitHub Pages，撰写参赛帖子

### 踩过的坑

1. **引力场性能**：p5.js 文字粒子数量过多会导致移动端卡顿。最终通过限制粒子数量、使用 requestAnimationFrame 优化、减少文字渲染频率解决。
2. **内容尺度**：发疯文学容易过界，需要反复调整"自嘲"和"轻生"的边界。最终原则是：可以拿"想死"开玩笑，但始终暗示"活着也挺好"。
3. **状态导向设计**：最初按功能模块划分页面，用户反馈"不知道该点哪个"。改为"我现在想..."的快速选择模式后，使用路径清晰很多。
4. **危机处理**：缓冲站需要分级引导。实现了轻度/中度/高危三级缓冲卡，高危级别直接提供紧急热线和 120/110 拨打按钮。
