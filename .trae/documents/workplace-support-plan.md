# Soul Soother 职场支持模块重构计划

> **目标**：将现有「心智障碍就业支持」模块重构为面向「情绪困扰人群的职场场景」的缓一缓空间
> **核心哲学**：不是转移抑郁焦虑，而是引导释放；不是工具堆砌，而是人格化陪伴
> **设计原则**：克制、有温度、有灵魂，让用户在某一秒觉得「好像不死也行」

---

## 一、现状分析

### 1.1 现有职场支持模块（位于「缓一缓」页面内）

| 功能 | 现状 | 问题 |
|------|------|------|
| 任务拆解 | 8种常见工作步骤拆解 | 面向心智障碍群体，对情绪困扰人群 irrelevant |
| 沟通卡片 | 5类15+句式一键复制 | 过于功能性，缺乏情感温度 |
| 工作节拍 | 4种番茄钟模式 | 纯工具，无陪伴感 |
| 感觉信号 | 6种emoji状态选择 | 表达工具，但无后续引导 |
| 紧急暂停 | 3种无声信号卡片 | 设计合理但内容单薄 |
| 工作提示 | 7条鼓励语随机显示 | 内容鸡汤化，缺乏真实感 |
| 公益招聘 | 6个职位+招聘渠道 | 与目标用户场景脱节 |

### 1.2 核心问题诊断

1. **人群错位**：原设计面向心智障碍就业者，现需面向情绪困扰的职场人群
2. **工具化过重**：功能像办公软件，缺乏「人格」和「陪伴感」
3. **内容鸡汤化**：鼓励语空洞，无法触达真实痛点
4. **缺乏释放出口**：没有引导用户「释放」情绪的设计，只有「管理」情绪的工具
5. **无连贯体验**：各功能孤立，没有形成「情绪上头→缓一缓→找到一点光」的闭环

## 二、用户洞察

### 2.1 目标用户画像

**典型场景**：凌晨1点，小李刚加班完，躺在床上刷手机。脑子里循环播放：「活着真没意思，明天还要上班，我受不了了」。他不想看医生（太麻烦），不想打电话（不想说话），不想被教育（「想开点」）。他打开 Soul Soother，想找到一个「让我缓一缓」的东西。

**核心特征**：
- 情绪上头时，理性下线，任何需要「操作」的东西都太麻烦
- 对「正能量」免疫，甚至反感
- 需要被「看见」，而不是被「解决」
- 在某一瞬间，一个恰到好处的句子、一个微小的互动，可能让他觉得「好像还能再撑一下」

### 2.2 情绪释放的理论基础

基于调研，有效的职场情绪释放方式包括：

| 释放方式 | 原理 | 设计转化 |
|----------|------|----------|
| **表达性书写** | 将情绪外化，降低心理负荷 | 职场发疯文学生成器 |
| **破坏性宣泄** | 安全范围内的「破坏」带来掌控感 | 虚拟捏泡泡、撕纸动画 |
| **身体释放** | 运动、呼吸、尖叫等生理调节 | 引导性身体动作 |
| **社会连接** | 知道「我不是一个人」 | 匿名同伴故事 |
| **意义重构** | 从「为什么是我」到「这有什么意义」 | 微意义发现工具 |
| **幽默消解** | 用荒诞对抗荒诞 | 职场梗图、自嘲文案 |

## 三、重构方案

### 3.1 整体架构：「职场缓一缓」空间

将原有7个功能整合为 **4个核心区域**，每个区域都有「小缓」的人格化引导：

```
职场缓一缓
├── 🌊 情绪释放区（原任务拆解+倒垃圾升级）
│   └── 引导用户「把情绪倒出来」而非「管理情绪」
├── 🤝 同伴故事区（原沟通卡片+陪伴卡升级）
│   └── 真实职场人的「我也经历过」故事
├── 🎯 意义微光区（原工作提示+留恋卡升级）
│   └── 帮助用户找到「今天值得的一个瞬间」
└── 🆘 紧急靠岸区（原紧急暂停+缓冲站升级）
    └── 当情绪淹没一切时的「最低限度生存指南」
```

### 3.2 区域一：情绪释放区 🌊

**设计目标**：让用户把情绪「倒出来」，而不是「压下去」

#### 3.2.1 职场发疯文学生成器（升级现有「倒垃圾」）

**交互流程**：
1. 用户输入一个关键词（如「加班」「老板」「KPI」）
2. 小缓以不同人格回应，生成针对性的发疯文学
3. 提供「复制」「截图」「再发疯一次」选项

**内容模板方向**：
- 加班系列：「轻舟已过万重山，低头一看是报表」
- 老板系列：「老板说要有格局，我只想有个觉睡」
- KPI系列：「KPI是老板画的饼，我是饼上的芝麻」
- 会议系列：「这个会开了三小时，结论是需要再开一个会」
- 通勤系列：「地铁挤成沙丁鱼，到了公司还要装成人」

**小缓人格介入**：
- 温柔小缓：生成后加一句「发泄完了，深呼吸一下」
- 毒舌小缓：「骂完了？骂得好。现在去喝口水。」
- 抽象小缓：「你的发疯文学已生成，建议发朋友圈仅自己可见。」

#### 3.2.2 虚拟宣泄工具

**捏泡泡纸**：
- 全屏泡泡纸，点击/滑动捏破
- 每捏破一个，弹出小缓的随机短句
- 音效：真实的泡泡纸声音

**撕纸动画**：
- 屏幕上有一张写着烦恼关键词的纸
- 用户滑动撕碎，碎片飘落
- 撕完后显示：「纸碎了，烦恼还在，但你发泄了，这很好」

**摔手机（虚拟）**：
- 屏幕显示一个手机模型
- 用户点击「摔」，手机掉落、屏幕碎裂动画
- 碎裂后显示：「手机没坏，你的心也不会。」

#### 3.2.3 身体释放引导

**3分钟办公室拉伸**：
- 简单的3-4个动作图示
- 每个动作30秒，配小缓语音引导文字
- 适合在工位/卫生间/楼梯间完成

**尖叫计数器**：
- 「如果你想尖叫，先在这里数10下」
- 点击数字倒数，配深呼吸引导
- 数完后：「好了，现在你可以选择真的去尖叫，或者发现没那么想尖叫了」

---

### 3.3 区域二：同伴故事区 🤝

**设计目标**：让用户知道「我不是一个人」，通过真实故事获得连接感

#### 3.3.1 「我也经历过」故事墙

**内容形式**：
- 匿名职场人的真实困境描述（1-2句话）
- 不展示「我是怎么走出来的」（避免说教感）
- 只展示「我现在还在这里，你也还在，这就够了」

**故事库方向**（30+条）：
- 「被裁员那天，我在公司楼下坐了三个小时，没哭，就是不想动。」
- 「连续加班一个月后，我在地铁上突然哭了，旁边的人假装没看见，我很感谢他。」
- 「述职报告前夜，我吞了半片安眠药，不是想死，只是想睡着。」
- 「老板说我效率低，我回工位后去洗手间吐了。」
- 「我每天上班前在车里坐15分钟，那是唯一属于我的时间。」
- 「辞职信写了七遍，第八遍终于发了。」

**交互设计**：
- 卡片式滑动浏览，像刷短视频
- 左滑「我也一样」（统计数字+1）
- 右滑「下一条」
- 点击「留一句」（匿名发布自己的一句话）

#### 3.3.2 「替我说」卡片（升级现有沟通卡片）

**重新设计方向**：
- 不是「如何与同事沟通」，而是「当我说不出口时，替我说」
- 面向的对象：同事、老板、HR、家人、自己

**卡片类别**：
| 对象 | 场景 | 示例 |
|------|------|------|
| 对老板 | 我无法完成 | 「这个任务我需要更多时间，或者需要支持。」 |
| 对同事 | 别打扰我 | 「我现在状态不好，晚点再聊可以吗？」 |
| 对HR | 我需要帮助 | 「我最近状态很差，公司有心理咨询资源吗？」 |
| 对家人 | 别问我工作 | 「今天不想聊工作，可以聊聊别的吗？」 |
| 对自己 | 我允许自己 | 「我今天可以不高效，可以不完美，可以只是存在。」 |

**交互**：
- 点击卡片 → 全屏展示 → 「复制」或「给TA看」
- 小缓会根据时间/场景推荐卡片（如深夜推荐「对家人」类）

### 3.4 区域三：意义微光区 🎯

**设计目标**：不是强行灌输意义，而是帮助用户发现「今天有一个瞬间值得」

#### 3.4.1 「今天的一个瞬间」微日记

**交互流程**：
1. 小缓问：「今天有没有一个瞬间，哪怕只有一秒，让你觉得『还好』？」
2. 提供选项：
   - 「有」（引导描述）
   - 「好像有，但想不起来了」（引导回忆）
   - 「完全没有」（进入「创造瞬间」模式）

**「创造瞬间」模式**：
- 小缓提供一个超小的行动建议：
  - 「去窗边看看云，拍下来，这就是今天的瞬间。」
  - 「给杯子倒满水，慢慢喝完，这就是今天的瞬间。」
  - 「打开音乐，听一首高中时的歌，这就是今天的瞬间。」
- 用户完成后，小缓：「你看，你创造了一个瞬间。明天也许还能再创造一个。」

#### 3.4.2 职场「小确幸」收集瓶

**设计**：
- 一个虚拟玻璃瓶，用户每记录一个「小确幸」，瓶子里多一颗星星
- 星星可以设置颜色、大小
- 瓶子满了可以「封存」，开启新瓶子

**小确幸示例**：
- 「今天地铁有座」
- 「咖啡比昨天好喝」
- 「同事没找我」
- 「准时下班了」
- 「外卖没洒」

#### 3.4.3 「我为什么还在这里」探索（升级留恋卡）

**重新设计**：
- 不是「你的猫在等你」这种通用文案
- 而是引导用户自己发现：「我还在这里，是因为______」

**交互**：
1. 小缓：「我不知道你为什么还在这里，但你知道。哪怕是一个很小的理由。」
2. 提供引导选项：
   - 「因为明天的早餐想吃那家包子」
   - 「因为那部剧还没看完」
   - 「因为有人还需要我」
   - 「因为我还没去过那个地方」
   - 「因为我答应过自己再试一次」
   - 「我真的不知道」（进入深度引导）

3. 用户选择或输入后，小缓：「这个理由很小吗？小也没关系。小的理由也是理由。」

### 3.5 区域四：紧急靠岸区 🆘

**设计目标**：当情绪淹没一切时，提供「最低限度的生存指南」

#### 3.5.1 「撑过下一分钟」生存指南

**设计原则**：
- 不提供「解决方案」，只提供「撑过去的方法」
- 每一步都极小、极具体、极可执行

**内容结构**：
```
你现在感觉撑不住了，对吗？
没关系。我们只做一件事：撑过下一分钟。

[1] 深呼吸（引导动画）
[2] 喝一口水（真实动作）
[3] 看看周围5样东西（ grounding ）
[4] 告诉自己：这种感觉会过去（文字展示）
[5] 选择：继续撑 / 需要人陪 / 需要专业帮助
```

#### 3.5.2 「陪我5分钟」陪伴模式

**设计**：
- 用户点击后，小缓开始5分钟陪伴
- 每分钟一条消息，不提问、不要求回复
- 内容示例：
  - 第1分钟：「我在这里。你不需要做任何事。」
  - 第2分钟：「深呼吸。吸气...呼气...」
  - 第3分钟：「你知道吗？你现在的感觉，很多人都有过。包括我。」
  - 第4分钟：「再撑一下。就一下。」
  - 第5分钟：「5分钟到了。你很棒。现在，你想做点什么吗？还是就这样待着？」

#### 3.5.3 职场专属求助资源

**重新整理资源**：
| 资源 | 场景 | 联系方式 |
|------|------|----------|
| 全国心理援助热线 | 任何时刻 | 400-161-9995 |
| 北京回龙观医院 | 危机时刻 | 010-82951332 |
| 生命热线 | 深夜陪伴 | 400-821-1215 |
| 职工心理咨询热线 | 职场专属 | 12351（工会） |
| 企业EAP（如有） | 在职员工 | 需用户自行填写 |

**设计**：
- 不直接展示号码（避免压迫感）
- 用户点击「我需要人陪」后，才展示
- 每个号码配一句话说明：「这个电话那头，有人愿意听你说。」

按场景分类：

- 职场冲突（8条）
- 职业倦怠（8条）
- 工作焦虑（8条）

### 4.3 小缓职场人格语录库

为每种小缓人格增加职场场景语录：

**温柔小缓**：
- 「工作不是你的全部，你的感受才是。」
- 「累了就休息，这不是偷懒，这是生存。」

**毒舌小缓**：
- 「老板画饼你还真吃？吐出来。」

**抽象小缓**：
- 「你的工位是租的，你的灵魂不是。」
- 「上班如上坟，但坟里埋的不是你。」

## 五、交互设计要点

### 5.1 人格化引导

每个区域都有小缓的「在场」：
- 进入区域时，小缓说一句引导语
- 用户操作时，小缓给予反馈
- 用户停留过久，小缓主动关心

### 5.2 低认知负荷

- 每个页面不超过3个主要操作
- 文字简短，一行不超过15字
- 优先使用选择而非输入

### 5.3 渐进式暴露

- 从「轻度释放」到「深度探索」渐进
- 不强制用户完成任何步骤
- 随时可退出，随时可返回

### 5.4 时间敏感

- 深夜（22:00-06:00）：优先展示「紧急靠岸」和「同伴故事」
- 工作时间（09:00-18:00）：优先展示「情绪释放」和「替我说」
- 周末：优先展示「意义微光」
