## 1. 项目概述

### 1.1 产品名称
**Soul Soother**（心灵抚慰者）

### 1.2 一句话描述
一个面向**心智障碍群体就业场景**的轻量级支持工具，通过任务拆解、沟通辅助、情绪支持和职场急救等功能，帮助用户在工作现场「缓一缓、稳住、继续」。

### 1.3 核心定位
- **不是职业评估工具**：明确免责声明，不做就业能力判断
- **是工作现场的小助手**：把复杂工作变成简单步骤，把难说的话变成可复制的卡片
- **是情绪安全网**：工作现场遇到困难时，随时可以「缓一缓」
- **是职场陪伴者**：让用户感受到「有问题没关系，有工具可以帮到你」

### 1.4 目标用户

#### 主要用户群
| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| **孤独症谱系障碍（ASD）** | 社交沟通困难、感官敏感、对变化适应慢 | 结构化任务分解、明确的沟通脚本、预见性提醒 |
| **智力发育迟缓** | 注意力有限、工作记忆较短、执行功能弱 | 简短指令、步骤可视化、即时正向反馈 |
| **唐氏综合征** | 语言表达较弱、社交意愿强但方式不当 | 简单明确的沟通辅助、情绪识别支持 |
| **脑瘫伴随智力障碍** | 肢体受限、表达困难、需要更多时间 | 无障碍操作设计、替代沟通方式、耐心引导 |

#### 辅助使用场景
| 使用方 | 场景 |
|--------|------|
| **就业辅导员** | 为个案制定工作计划时作为参考工具 |
| **雇主/同事** | 了解如何与心智障碍员工有效沟通 |
| **家长/家属** | 在家庭支持中延伸使用工作相关的练习 |

## 2. 需求分析

### 2.1 用户痛点

#### 核心痛点矩阵

| 痛点维度 | 具体表现 | 对工作的影响 | 现有解决方案不足 |
|----------|----------|--------------|------------------|
| **表达困难** | 不知道如何向同事/领导表达需求 | 遇到问题时不敢求助，容易积累压力 | 没有适合的「替我说」工具 |
| **任务理解** | 多步指令记不住、顺序容易混乱 | 工作效率低、容易出错、产生挫败感 | 缺乏「一步一步来」的拆解工具 |
| **情绪调节** | 感到紧张/焦虑时不知道如何处理 | 容易情绪失控、影响工作表现 | 情绪调节工具与工作场景脱节 |
| **环境适应** | 新环境、新任务容易焦虑 | 换岗、换领导时适应困难 | 缺乏预见性引导和过渡支持 |
| **社交互动** | 不知道怎么开始对话、不知道怎么拒绝 | 被同事误解、被动接受过多任务 | 缺乏「说什么、怎么说」的参考 |
| **自我认知** | 不确定自己的状态是好是坏 | 无法主动求助，容易硬撑 | 缺乏简单自我状态评估工具 |

### 2.2 典型工作场景

#### 场景一：接到新任务时的恐慌
> **小星（孤独症）**在咖啡厅工作，这天店长让她去整理新到货的纸杯。
>
> 她打开 Soul Soother 的「任务拆解」，选择「整理桌面/整理物品」，看到清晰的两步指引：
> 1. 把东西分类：要的 / 不要的
> 2. 要的东西放回原位
>
> 她按照步骤一步一步做，完成后感觉「今天又搞定了一件事」。

#### 场景二：需要求助但不知道怎么开口
> **小明（智力发育迟缓）**在工厂流水线工作，机器突然发出了奇怪的声音。
>
> 他打开「沟通卡片」，选择「求助」类别，点击了第一张卡片「我遇到困难了，可以帮我一下吗？」
>
> 他拿着手机给班长看，班长立刻过来帮忙处理了问题。

#### 场景三：工作节奏太快感到压力
> **小月（唐氏综合征）**在超市整理货架，感到周围太吵、人太多，有些焦虑。
>
> 她打开「工作节拍」，选择了「轻松节奏」（10分钟工作/2分钟休息），开始跟着计时器工作。
>
> 每当计时器响起，她就停下来深呼吸，10分钟后继续。这样她能坚持完成一上午的工作。

#### 场景四：感到快要失控时
> **小文（脑瘫伴随智力障碍）**在办公室做简单的文件整理，感到有人一直在旁边说话，越来越烦躁。
>
> 他点击「感觉信号」中的「😣 很紧张」，看到提示「需要深呼吸或暂停」。
>
> 他又点击「紧急暂停」中的「🛑 我要暂停」，给旁边的同事看，同事理解地走开了。

#### 场景五：深夜加班/早班前的焦虑
> **小辰（孤独症）**明天要第一次参加早会，很担心自己会出错。
>
> 睡前他打开 Soul Soother，看了看「留恋卡片」：「明天的事明天再说，今晚先睡好」。
>
> 他又看了「工作提示」：「做不完没关系，分成几次做就好」。
>
> 他感觉安心了一些，关灯睡觉。

## 3. 核心设计理念

### 3.1 设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **简单到不能再简单** | 每个操作不超过3步，每个步骤不超过10个字 | 「打开 → 选择 → 完成」 |
| **看得见摸得着** | 用图标、emoji、数字编号代替抽象描述 | 用 😊😓😰 代替「情绪状态」 |
| **可以复制粘贴** | 沟通卡片一键复制，拿到直接用 | 「我遇到困难了，可以帮我一下吗？」 |
| **永远不评判** | 不说「你应该」「你必须」，只说「可以」「试试」 | 「可以休息一下」而不是「你太紧张了」 |
| **安全网永远在** | 任何时候都能找到紧急暂停/求助入口 | 底部常驻紧急入口 |

### 3.2 针对心智障碍群体的设计考量

#### 3.2.1 认知负荷控制
- **信息量控制**：每次只呈现3-5个选项，避免选择过载
- **视觉层级**：用大小、颜色、位置区分重要程度
- **一致的布局**：同类功能放在固定位置，降低学习成本
- **明确的反馈**：每个操作都有即时反馈（声音、动画、颜色变化）

#### 3.2.2 感官友好设计
- **低饱和度配色**：避免刺眼的纯色，减少视觉刺激
- **可关闭音效**：所有声音可静音，尊重听觉敏感者
- **简洁动画**：动画缓慢柔和，避免快速闪烁
- **足够的触控区域**：按钮至少48x48px，方便点击

#### 3.2.3 沟通辅助设计
- **替代沟通系统（AAC）理念**：用图标/文字代替口头表达
- **可复制的句子**：不用现场组织语言，直接用现成的
- **分类清晰**：沟通内容按场景分类（求助/休息/确认/感谢）
- **可展示设计**：紧急卡片可以放大展示给旁人看

#### 3.2.4 自主性支持
- **用户主导**：用户自己选择想用哪个功能，不被强制安排
- **可预测**：进入后的内容布局可预测，减少不确定性
- **可退出**：任何时候都可以退出或返回，没有「必须完成」的压力
- **尊重节奏**：计时器可以暂停、可以重置

### 3.3 语言风格指南

| 场景 | 传统方式（❌） | Soul Soother 方式（✅） |
|------|--------------|------------------------|
| 用户表达困难 | 「你有什么想说的可以告诉我」 | 「点击卡片，直接拿给同事看」 |
| 用户任务出错 | 「没关系，下次注意」 | 「出错也没关系，可以改正」 |
| 用户感到焦虑 | 「放松点，别紧张」 | 「感觉紧张是正常的，我们有办法」 |
| 用户需要休息 | 「休息一下也好」 | 「累了就休息，这不是偷懒」 |
| 用户求助被拒绝 | 「没关系，再试试」 | 「换个人问，或者等会儿再试」 |

## 4. 功能规划

### 4.1 页面结构

```
Soul Soother
├── 首页 (Home)           # 引力场词汇交互 + 紧急入口
├── 快速检测 (Checkin)    # 小缓对话式情绪检测
├── 陪伴 (Mutual)         # 陪伴卡、弹幕墙、留言
├── 缓冲站 (Kit)          # 呼吸空间、缓冲卡、资源
└── 缓一缓 (Play)         # 喘口气、抓回现实、等一下、找理由...
    └── 职场支持 (Work)    # 【命题一定制】任务拆解、沟通卡片、工作节拍、感觉信号、紧急暂停
```

### 4.2 功能详解

#### 4.2.1 首页 - 引力场词汇交互
- **背景词汇**：漂浮的词汇分为积极词汇（彩色）和丧词汇（灰色）
- **点击积极词汇**：词汇被吸引到屏幕中央，产生粒子特效
- **点击丧词汇**：词汇淡出消失，生成积极词汇替代
- **紧急入口**：底部固定紧急入口，随时可进入缓冲站

#### 4.2.2 快速检测 - 小缓对话式
- **对话形式**：AI角色"小缓"以聊天形式进行情绪检测
- **问题类型**：选择题、滑动条评分
- **即时反馈**：根据用户回答给出温柔回应
- **结果建议**：检测结束后提供个性化建议

#### 4.2.3 陪伴页面
- **陪伴卡抽取**：点击随机获取一张温暖的陪伴卡片
- **深夜弹幕墙**：实时滚动的匿名留言
- **留一句给陌生人**：用户可以发布匿名留言

#### 4.2.4 缓冲站页面
- **30秒呼吸空间**：倒计时呼吸练习
- **缓冲卡**：三个等级（轻度/中度/高危）的应对建议
- **温暖资源**：心理援助热线、医院联系方式

#### 4.2.5 缓一缓页面
- **喘口气**：呼吸动画引导深呼吸
- **抓回现实**：五感 grounding 练习
- **等一下**：延迟决定清单
- **找理由**：留恋卡片
- **倒垃圾**：发疯文学生成器
- **翻译情绪**：把坏念头翻译成真实需求
- **玩个小游戏**：推荐放松小游戏

#### 4.2.6 职场支持（命题一定制功能）⭐

##### 4.2.6.1 任务拆解
**功能说明**：把常见工作拆成1-2步，用数字编号和简单语言呈现。

**任务类型**：
| 任务 | 步骤 |
|------|------|
| 整理桌面/物品 | 1. 把东西分类：要的 / 不要的<br>2. 要的东西放回原位 |
| 发邮件 | 1. 打开邮箱，写收件人<br>2. 写标题和内容，发送 |
| 接电话 | 1. 说"你好"<br>2. 听对方说什么，需要时记下来 |
| 参加会议 | 1. 找到会议室，坐下来<br>2. 听别人说，需要时点头 |
| 喝水休息 | 1. 站起来走到饮水机<br>2. 倒水，喝完回来 |
| 去洗手间 | 1. 站起来走到门口<br>2. 找洗手间标志 |
| 打印文件 | 1. 把文件发送到打印机<br>2. 去打印机那里拿 |
| 扫地 | 1. 拿扫帚和簸箕<br>2. 从里往外扫，倒进垃圾桶 |

**交互设计**：
- 点击任务卡片 → 展开详细步骤
- 点击每个步骤 → 标记为完成（✅）
- 全部完成 → 获得正向反馈

**设计考量**：
- 步骤用数字编号，帮助理解执行顺序
- 每步只用一句话，避免信息过载
- 可点击反馈，满足即时反馈需求

##### 4.2.6.2 沟通卡片
**功能说明**：预设职场常用句子，一键复制，直接给同事看或展示。

**卡片类别**：
| 类别 | 图标 | 示例句子 |
|------|------|----------|
| 求助 | 🆘 | "我遇到困难了，可以帮我一下吗？"<br>"这个我不会，能教我吗？"<br>"我需要一点时间想想" |
| 需要休息 | ⏸️ | "我需要休息一下"<br>"我去倒杯水"<br>"我出去走一走" |
| 确认理解 | ❓ | "我理解对了吗？"<br>"可以再说一遍吗？"<br>"请问是这个意思吗？" |
| 表达感谢 | 🙏 | "谢谢你！"<br>"帮了大忙了"<br>"很高兴你帮我" |
| 开始/结束 | 👋 | "我开始工作了"<br>"我准备好了"<br>"我今天先到这里" |

**交互设计**：
- 点击类别标签 → 筛选显示该类别的卡片
- 点击卡片 → 一键复制到剪贴板
- 复制成功 → 显示反馈提示

**设计考量**：
- 句子简短（不超过15字），便于记忆和复述
- 可展示设计：可以拿手机给旁人看
- 图标辅助识别，减少文字依赖
- 可分类筛选，避免信息过载

##### 4.2.6.3 工作节拍（番茄钟）
**功能说明**：可视化工作/休息节奏，帮助维持专注和避免过度疲劳。

**节拍模式**：
| 模式 | 工作时间 | 休息时间 | 适用场景 |
|------|----------|----------|----------|
| 番茄钟 | 25分钟 | 5分钟 | 常规工作节奏 |
| 短时专注 | 15分钟 | 3分钟 | 初次适应阶段 |
| 长时专注 | 45分钟 | 10分钟 | 稳定工作期 |
| 轻松节奏 | 10分钟 | 2分钟 | 高焦虑/低能力期 |

**交互设计**：
- 点击模式卡片 → 选择工作节拍
- 点击「开始」→ 计时器倒计时
- 工作结束 → 提示休息，播放提示音
- 休息结束 → 提示继续工作

**视觉设计**：
- 圆形进度环，直观显示剩余时间
- 颜色区分工作状态（红色）和休息状态（绿色）
- 数字大而清晰，方便阅读

**设计考量**：
- 多档位可选，尊重个体差异
- 可暂停可重置，没有强迫感
- 明确的开始/暂停状态，减少焦虑
- 到时自动提醒，不需要持续关注

##### 4.2.6.4 感觉信号
**功能说明**：用emoji图标表达当前状态，帮助他人理解自己的需求。

**信号类型**：
| 状态 | emoji | 标签 | 提示信息 |
|------|-------|------|----------|
| 良好 | 😊 | 我很好 | 状态好，可以继续工作 |
| 疲惫 | 😓 | 有点累 | 需要小休息，喝点水 |
| 压力 | 😰 | 太多了 | 需要把任务减少一点 |
| 困惑 | 🤔 | 我不懂 | 需要有人解释一下 |
| 紧张 | 😣 | 很紧张 | 需要深呼吸或暂停 |
| 求助 | 🆘 | 需要帮助 | 请人来帮忙 |

**交互设计**：
- 点击图标 → 选中状态
- 选中状态 → 显示提示信息
- 可多选 → 同时表达多种感受

**设计考量**：
- 用emoji代替抽象描述，直观易懂
- 点击即选，操作极简
- 可展示给旁人看，作为无声沟通工具
- 提示信息告诉旁人应该怎么帮忙

##### 4.2.6.5 紧急暂停
**功能说明**：需要立刻暂停时，向同事/领导无声展示的信号。

**紧急信号**：
| 信号 | emoji | 标签 | 说明 |
|------|-------|------|------|
| 暂停 | 🛑 | 我要暂停 | 表示需要暂停一下 |
| 离开 | 🏃 | 我要离开 | 表示需要出去一下 |
| 求助 | 📞 | 打电话 | 表示需要打电话求助 |

**交互设计**：
- 点击信号卡片 → 显示「请把这个给同事看」提示
- 卡片高亮显示3秒 → 可直接展示手机屏幕

**设计考量**：
- 专为「开口困难」设计，无需口头表达
- 可无声展示，减少社交压力
- 3秒后自动恢复，不造成尴尬
- 作为情绪失控前的「预防性」工具

##### 4.2.6.6 工作提示
**功能说明**：随机显示温暖的工作支持提示。

**提示示例**：
- "做不完没关系，分成几次做就好"
- "有困难可以问同事，不要自己硬撑"
- "累了就休息，这不是偷懒"
- "做错事也没关系，可以改正"
- "今天的任务做不完，明天继续"
- "慢一点没关系，稳稳地做就好"
- "完成一小步就是进步"

**设计考量**：
- 肯定性语言，强调进步而非完美
- 符合目标用户的认知水平
- 可重复查看，强化正向思维

##### 4.2.6.7 公益招聘信息
**功能说明**：整合公益性招聘信息源，帮助心智障碍群体找到适合的工作机会。

**工作类型分类（基于需求维度）**：

| 维度 | 类型 | emoji | 说明 | 适合人群 |
|------|------|-------|------|----------|
| **环境** | 安静环境 | 🔇 | 不需要说话，安静独立工作 | 对声音敏感、不擅长社交 |
| | 接触自然 | 🌿 | 户外工作，接触花草植物 | 喜欢安静、需要放松 |
| | 固定场所 | 🏠 | 室内工作，有固定工位 | 喜欢稳定、不喜欢变化 |
| **社交** | 独立工作 | 🧩 | 一个人完成，不用和很多人说话 | 不擅长社交、容易紧张 |
| | 轻度互动 | 🤝 | 偶尔说话，不需要主动推销 | 能进行简单交流 |

## 5. 技术方案

### 5.1 技术栈
- **前端**：HTML5 + CSS3 + JavaScript（纯原生，零框架）
- **动画**：p5.js（引力场词汇效果）
- **AI集成**：支持多种AI模型（NVIDIA、通义千问、Kimi等）
- **存储**：localStorage（对话记忆、配置）
- **音效**：Web Audio API

### 5.2 项目结构

```
soul-soother/
├── index.html              # SPA 单页应用入口（包含内联CSS和JS）
├── css/
│   └── style.css           # 样式文件（已合并到index.html）
├── js/
│   ├── app.js              # 主应用逻辑（已合并到index.html）
│   ├── data.js             # 内容数据
│   ├── gravity.js          # 引力场词汇逻辑
│   └── p5-gravity.js       # p5.js 引力场实现
├── audio/                  # 音效文件
└── README.md
```

### 5.3 响应式设计
- **移动端优先**：针对手机屏幕优化（主要使用场景）
- **触控友好**：大按钮、大触控区域
- **横屏支持**：平板设备支持横屏使用

### 5.4 无障碍设计
- **语义化HTML**：使用正确的标签和角色
- **颜色对比度**：符合WCAG 2.1 AA标准
- **字体可缩放**：支持系统字体大小设置
- **键盘可操作**：所有功能可通过键盘访问
